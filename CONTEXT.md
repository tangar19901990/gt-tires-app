# CONTEXT — правила проєкту GT Tires CRM

Цей файл — звід правил для роботи над кодом (для себе і для будь-якого AI-помічника).
Прочитати **перед** будь-якою правкою.

---

## 1. Головний принцип: «хірург, не трактор»

Зміни — **точкові й адитивні**. Додаємо нове поряд зі старим, не переписуємо робоче.
Краще новий блок/функція, ніж рефакторинг наявного. Якщо правка зачіпає багато місць —
зупинись і переузгодь.

---

## 2. Що НЕ чіпати

- **Назви функцій** — не перейменовувати. Їх викликають inline-обробники (`onclick=...`),
  будь-яке перейменування ламає UI мовчки.
- **Друк** (модуль `PRINT`, шаблон рахунку з вшитим CSS) — не торкатися.
- **Supabase** (`SUPABASE SYNC`, ключі, синхронізація) — не торкатися.
- **AI Tools** (усі секції `🧠 AI TOOLS`, `🤖 OpenRouter`, `🛞 MARKET AGENT`) — не торкатися.
- **TOP SECRET / конфіденційні бізнес-поля** — не виносити в публічні доки/логи/коміти.

---

## 3. Архітектура коротко

- Один HTML + `css/styles.css` + `js/app.js`. Без збірки, без npm-залежностей у рантаймі.
- `app.js` — **класичний скрипт** (НЕ `type="module"`). Уся область видимості глобальна,
  тому функції лишаються глобальними навмисно.
- Порядок завантаження критичний: розмітка → Supabase CDN → `app.js` (в кінці `<body>`).
  Init-IIFE наприкінці `app.js` чекає, що DOM уже існує. Скрипт не переносити в `<head>`.
- Стан застосунку — у `localStorage`, префікс ключів **`gt_tires_v4_`**.
  Зчитування/запис через хелпери `load/save/loadVal/saveVal` (вгорі секції `DATA`).
- Синхронізація: `gtAutoSync` кожні 30 с (offline-first, additive).
- Мова інтерфейсу — **українська**. Нові рядки UI — українською.

---

## 4. Робочий цикл правки

1. **Backup** робочого файлу в `backup/` з датою.
2. Правка — точкова, бажано окремий блок/функція.
3. `node --check js/app.js` — синтаксис.
4. Перевірити, що HTML підключає `css/styles.css` і `js/app.js`.
5. Прогнати тести (jsdom-набір), якщо доступний.
6. Візуальна перевірка на мобілці та десктопі.
7. Коміт із коротким описом що саме додано.
