# PROJECT STRUCTURE

Карта проєкту GT Tires CRM. Номери рядків — усередині `js/app.js`
(станом на розділення, 3441 рядок, 267 функцій-декларацій).

---

## Дерево файлів

```
.
├── gt-tires-v4-supabase-ready.html   розмітка; <link css> у <head>, <script app.js> перед </body>
├── css/
│   └── styles.css                    усі стилі застосунку
├── js/
│   └── app.js                        уся логіка (vanilla JS, класичний глобальний скрипт)
├── backup/
│   └── gt-tires-v4-supabase-ready_РРРР-ММ-ДД.html
├── docs/
│   └── PROJECT_STRUCTURE.md           (цей файл)
├── README.md
└── CONTEXT.md
```

---

## Порядок завантаження (важливо)

```
1. <meta charset="UTF-8">  +  Google Fonts
2. Supabase CDN  (рядок 12 HTML)
3. розмітка <body>
4. css/styles.css  → через <link> у <head>
5. js/app.js       → перед </body> (рядок ~342)
       └─ init-IIFE наприкінці app.js спрацьовує, коли DOM уже готовий
```

`app.js` — НЕ модуль. Усі функції глобальні (їх викликають `onclick`/`onchange` у розмітці).

---

## Модель даних

- Сховище: `localStorage`, префікс `gt_tires_v4_`.
- Хелпери (секція `DATA`): `load`, `save`, `loadVal`, `saveVal`, `uid`, `fmtDate`, `fmtMoney`, `today`.
- Основний стан: `orders`, `clients`, `tires`, `warehouse`, `cashbook`,
  `usedTires`, `marketPrices`, `aiLogs`, `customServices`, `quickServices`.
- Синхронізація: `gtAutoSync` кожні 30 с (offline-first, секція `SUPABASE SYNC`).
